Tanuj Garg, Alok Kumar Kar, Mashrukh Islam, Saurabh Chand

Hate Speech is commonly defined as any communication that disparages a person or a group on the basis of some characteristic such as race, color, ethnicity, gender, sexual orientation, nationality, religion, or other characteristics. With the rise of social media and user generated content, detecting and classifying hate speech is becoming quite important. All platforms do have some form of manual checking for this but it can’t be as fast as automated classification.

Problem Statement

We will be identifying if the target of hate is a single human or a group of persons and if the message author intends to be aggressive, harmful, or even to incite, in various forms, to violent acts against the target, specifically on social media platforms.

  • Task 1: Classify hateful tweets (where Hate Speech against women or immigrants has been identified) as aggressive or not aggressive. Note that aggressiveness is not a mandatory characteristic for all hateful texts and some text can express hate against a target in terms of disrespect without using an aggressive language.
  • Task 2: Classify hateful tweets to identify if the target harrased is a generic group of people or a specific individual.

Dataset

The dataset is a twitter dataset which was provided to us.

  • 9000 training set tweets
  • 1000 validation set tweets
  • 3783 tweets labelled as hateful.
  • 5217 tweets labelled as not hateful.

Approach

  1. Preprocessing :-
    The first big challenge was to clean and process the tweets to remove the noise and other unnecessary words, which would be quite useful for training any model and would be helpful in producing better results.
    • Remove_URL: process tweets with and without removal of URL.
    • Remove_Hashtags: process tweets with and without removal of Hashtag
    • Remove_Swords: process tweets with and without removal of Stopwords
    • Stem_tweet: process tweets with and without stemming of words using Snowball stemmer
    We found that the accuracy of various models change when we use the above functions to modify the tweets, in general on a positive side. Preprocessing procedure depends upon the model being used, we used all of the above functions for training our model.
    This part was done using regular expressions to clean the tweet and select which parts of the tweets should be there, e.g. mentions, urls,etc. We didn’t use any external libraries for this part. Different results were obtained depending on which subset of things were kept e.g. mentions, urls, emojis.
  2. Data Augmentation :-
    As the data provided to us was just 10000 tweets, we decided on using Data Augmentation in order for our model to learn more from the data. We used two ways to augment the data.
    • EDA : We did the four following operations on a training example to create a new training example
      • Synonym Replacement : Replace a random word with its synonym
      • Random Insertion : Insert synonym of a random word at random place
      • Random Swap : Randomly choose two words and swap them
      • Random Deletion : Randomly remove each word with probability p=0.1
      Newly generated tweet is different from the original one. Some of these operations introduce some noise in data which makes the model more robust
    • Machine translation : We convert the english tweets to some other languages such as french, german etc. Then we convert the tweets back to english. This method preserves the sentiment of the tweets but changes the words used in the tweet which results in a different new tweet altogether.
  3. Convolutional Neural Networks :-
    Initially, the Convolutional Neural Network architecture was designed for image processing, however it has been commonly used in the sentiment analysis. CNN’s are good at extracting local and position-invariant features. For tasks where feature detection in text is more important, for example, searching for angry terms, sadness, abuses, named entities etc. CNN’s work well which is why CNN is chosen as a model here for the tasks given. The model is as follows :
    • After the data is processed and padded and converted to a word embedding matrix it is fed into the embedding layer which requires the input data be integer encoded so that each word is represented by a unique integer. The Embedding layer is initialized with random weights and will learn an embedding for all of the words in the training dataset. The output of the Embedding layer is a 2D vector with one embedding for each word in the input sequence of words.
    • Feature Extraction : In the feature extraction step only two layers of convolution and two layers of pooling were used, with tanh activation function. Four filters were used, 2 of 2 dimensions and 2 of 3 dimensions. Each filter refers to the classic n-grams technique (extremely used in bag-of-words based models), which consists of processing a group of n words, in order to consider not only isolated words in a tweet, but also the context in which they are inserted. The filters are applied under the vector representation of the input tweets (embedding layer), using the concept of Back propagation to adjust the weights dynamically. These filters can extract lexical, syntactic or semantic features automatically. Finally, the two layers of convolution and pooling are concatenated and directed to the next step.
    • In the classification stage, we used two dense layers, the first one has 512 neurons, relu function of activation and dropout of 0.5. The second has a neuron and sigmoid activation function, phase where classification occurs. For the training, the loss and optimization functions used were binary cross entropy and RMSprop (with learning rate 0.001) respectively.
    • This model is the direct implementation of the architecture proposed by Zhang and Wallace,2015.
    Two different methods after this :
    • Three classes were formed - Hate-Aggressive/Target, Hate-Non Aggressive/Target and Non-Hate and loss was changed to categorical crossentropy to amount for the multiple classes. However this method fares very badly due to the huge imbalance in the three classes - 2:2:5 respectively, and ultimately this method was dropped from consideration.
    • The other method consists of :
      • At the first level - The model is trained to detect hateful tweets.
      • At the second level - Only hateful tweets are taken and they are subdivided into aggressive/non-aggressive and individual/group which are again fed into the model and classified.
      • This method performs considerably better than the first method due to much better class numbers.

Evaluation Metric

As the tasks involve us doing binary classification we don’t have a negative class at all. Hence we decided to use Accuracy as our evaluation metric.

Results

The following are the best results that were obtained and results after applying data augmentation.

  1. Naive Bayes Model
    • Aggression Detection :
      65.25% accuracy, without stemming, without cleaning emojis and mentions & without including length as a feature
    • Target Identification :
      89.6% accuracy, after cleaning out emojis but without stem, without cleaning emojis and without including length as a feature
  2. Logistic Regression Model
    • Aggression Detection :
      65.25% accuracy, after stemming and cleaning mentions but without cleaning emojis and without including length as a feature
    • Target Identification :
      88.89% accuracy, after cleaning mentions, but without stemming, without cleaning emojis and without including length as a feature
  3. Support Vector Machine Model
    • Aggression Detection :
      60.76% accuracy, after stemming and cleaning emojis & mentions, but without including length as a feature
    • Target Identification :
      81.8% accuracy, without stemming, without cleaning out emojis & mentions and without including length as a feature
  4. Convolutional Neural Network Model
    • Hate Speech Identification :
      73.5% accuracy, without stemming, hashtag removal, url removal and cleaning, using Common Crawl Glove 300d embeddings
    • Aggression Detection :
      Classifying Hateful tweets first and then using hateful tweets to predict Aggressive tweets
      67.21% accuracy, without stemming, hashtag removal, url removal and cleaning, using FastText Embeddings
    • Target Identification :
      Classifying Hateful tweets first and then using hateful tweets to predict Target : Individual or Group
      89.93% accuracy, without stemming, hashtag removal, url removal and cleaning, using Common Crawl Glove 300d embeddings
  5. Long short-term memory Model
    • Aggression Detection :
      Classifying Hateful tweets first and then using hateful tweets to predict Aggressive tweets
      59.7% accuracy
    • Target Identification :
      Classifying Hateful tweets first and then using hateful tweets to predict Target : Individual or Group
      79.62% accuracy

Analysis

We found that CNNs give the best results here compared to LSTMS as LSTMs usually are good at predicting what comes next in a sequence while CNNs can learn to classify a sentence or a paragraph; hence the observation. CNN is looking for the same patterns over different regions of text. For example, one can use a CNN with a softmax classifier at the end, to classify a piece of text - say for sentiment analysis. While LSTMs solve the inherent problems in RNNs, they still are not better than a pure CNN for classification. CNNs are inherently faster than and much less computationally expensive compared to RNN and LSTM which are difficult to train because they require memory-bandwidth-bound computation.
